# Pokémon Console Battle

A terminal-based Pokémon battle simulator featuring OOP logic and ASCII Art. Built for C# study and practice purposes.

## -Key Features

- **Dynamic Selection:** Interactive menu using arrow keys to choose your Pokémon and your opponent.
- **Real-time Sprites:** The interface toggles between a Pokéball and the Pokémon's ASCII sprite during navigation.
- **Combat System:** Turn-based mechanics (Player vs. CPU) with damage processing and varied move sets.
- **Status Effects:** Implementation of Burn, Poison, and Paralysis, each with its own turn-based damage logic and duration.
- **Visual Interface:**
    - Dynamic HP bars that update in real-time.
    - Visual feedback (flashing) when a Pokémon takes damage.
    - Progressive loading bar with percentage markers between game phases.
- **Layout Management:** Margin and padding system to ensure the UI remains centered and organized within the console.

## -Applied Programming Concepts

- **Object-Oriented Programming (OOP):** Organized entities (Pokémon, Moves) into classes and objects.
- **State Management:** Turn-based flow control and duration counters for passive effects.
- **Console Interactivity:** Capturing `ConsoleKey` input for menu navigation instead of typed commands.
- **Timing Logic:** Utilizing `Thread.Sleep` to manage game pacing and smooth transitions.
